# Direct raylib 6 bindings for Rust

This is fork and continuation of `raylib-ffi` that  aims to provide more complete and up to date direct binding to [raylib](https://www.raylib.com/) for rust developers.

This library is largely auto-generated by the code in `build/`. Raylib itself exposes lots of metadata, meaning that wrappers can auto-translate the C API into whatever language is needed (in this case, Rust).

It also tries to provide easy compiling for wasm target.

For wasm compilation you need to have properly configured [emscripten SDK](https://emscripten.org/).

## Dependencies

### Fedora:

### Windows
- You need up to date Rust (duh)
- Cmake (you may want to use scoop for example)
- For Wasm comoilation you need emscripten SDK, you can get it from [emscripten official site](https://emscripten.org/)

## Verifying your build

`raylib-ffi` bundles a rust version of the example project from raylib proper. To verify your build of this library worked, run:

```sh
cargo run --example basic
```
